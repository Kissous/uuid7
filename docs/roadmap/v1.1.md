# v1.1.0 — Same-millisecond monotonicity

Guarantees that UUIDs generated within the same millisecond are strictly increasing.

- **Status**: in development
- **Goal**: monotonic generation per RFC 9562, with no public API change

## Scope

Only the internals of `Uuid7::generate()` change. The public API is identical to
v1.0 — no new method, no signature change.

## Behavior

- Two UUIDs generated in the same process are **strictly increasing**, even when
  produced within the same millisecond.
- Monotonicity also holds if the system clock moves **backwards** (the previous
  timestamp is reused rather than emitting a smaller value).

## Technical details

- RFC 9562 "monotonic random" method: the 74-bit (`rand_a` 12 bits + `rand_b`
  62 bits) field is seeded randomly on each new millisecond and incremented as a
  counter for subsequent calls in the same millisecond.
- Counter overflow within one millisecond (2^74 values — unreachable in practice)
  rolls over into the next millisecond with fresh randomness.
- State is **process-local static** state. PHP is shared-nothing per request, so
  ordering is guaranteed within a single process. Across separate processes,
  timestamps separate values and a 74-bit random collision is negligible.
- Still `random_bytes()` only; version (7) and variant (RFC 9562) bits unchanged.
