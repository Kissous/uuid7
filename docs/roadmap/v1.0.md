# v1.0.0 — Minimal UUIDv7

First stable release: minimal but production-ready.

- **Status**: released
- **Goal**: generate, parse and validate UUIDv7 values per RFC 9562

## Scope

Strictly limited to generating, parsing and validating UUIDv7 values.
No other feature must be added in this version.

## Public API

| Method | Description |
|--------|-------------|
| `Uuid7::generate(): self` | Generates a UUIDv7 (ms timestamp + `random_bytes` randomness). |
| `Uuid7::fromString(string $uuid): self` | Parses a canonical form; throws `InvalidUuidException` if invalid. |
| `Uuid7::toString(): string` | Canonical 8-4-4-4-12 form, normalized to lowercase. |
| `Uuid7::__toString(): string` | Same via `Stringable`. |

### Exceptions

| Exception | Thrown when |
|-----------|-------------|
| `Exception\InvalidUuidException` | `fromString()` receives a non-conforming string (length, hex, version ≠ 7, non-RFC variant, missing hyphens). |

## Technical details

- RFC 9562-compliant 128-bit layout: 48-bit timestamp (big-endian),
  4-bit version (`0b0111`), 12-bit random, 2-bit variant (`0b10`), 62-bit random.
- Internal source of truth: 16 raw bytes (prepares `toBytes`/`toHex` in v1.3).
- Randomness exclusively from `random_bytes()` (never `rand`/`mt_rand`/`uniqid`).
- `fromString()` is case-insensitive; `toString()` always returns lowercase.
- `final`, immutable value object (private constructor), implements `Stringable`.

## Tests

- **Unit** (`tests/Unit/Uuid7Test.php`): format, version, variant, uniqueness,
  `fromString`/`toString` round-trip, case normalization, `Stringable` cast,
  rejection of invalid input (wrong version, wrong variant, non-hex, missing hyphens).
- **Property**: to be completed (monotonicity arrives in v1.1; uniqueness at volume here).
- **Conformance**: validate at least one RFC 9562 sample vector.
